# Agile Workflow Guide for FreshDish

This guide explains how to use Agile practices to manage your FreshDish project efficiently, without changing your codebase.

## 1. Organize Your Work

### Using GitHub Projects (Step-by-Step for Solo Developers)
1. Go to your repository on GitHub.
2. Click on the "Projects" tab (or "Projects" in the sidebar).
3. Click "New project" and choose "Board".
4. Name your project (e.g., FreshDish Tasks).
5. Add columns: To Do, In Progress, Done (you can add more if you like).
6. Click "Add cards" to create tasks for each feature, bug, or idea.
7. As you work, drag cards from "To Do" to "In Progress" and then to "Done".
8. You can link cards to issues or pull requests if you want.

This helps you visually track your progress and stay organized, even if you’re working alone.

- Use a task board (like GitHub Projects) with columns:
  - **Backlog**: Ideas and features to do
  - **To Do**: Tasks ready to start
  - **In Progress**: Tasks being worked on
  - **Review/Testing**: Tasks ready for review
  - **Done**: Completed tasks

## 2. Break Down Features
- Write user stories for each feature (e.g., "As a user, I want to add items to my cart so I can order food").
- Split big features into smaller tasks (frontend, backend, testing, etc.).

## 3. Plan Sprints
- Choose a sprint length (1–2 weeks is common).
- At the start of each sprint, select tasks from the Backlog to move to To Do.

## 4. Daily Standups
- Meet briefly each day to discuss:
  - What was done yesterday
  - What will be done today
  - Any blockers

## 5. Review & Retrospective
- At the end of each sprint:
  - Demo completed features
  - Discuss what went well and what can improve

## 6. Adapt & Repeat
- Move unfinished tasks back to To Do or Backlog.
- Start the next sprint!

---

**Tip:** Use GitHub Issues and Projects for a simple, integrated Agile board.

This guide is for your team’s workflow only—no code or features are changed by following it!
